User Stories

Guest

Application Information
  • I need to see instructions on how to use the application.
  • I need to be directed to a landing page upon visiting the website where I can learn more about the application.
Privileges
  • I need to be able to create an account on GameEye and become a registered user.
  • I must not be able to access other user accounts.
  • I must not be able to use GameEye unless registered.

Registered User

Account Management & Login
  • I need to be able to login.
  • I need to create an account using my email as a username.
  • I need to be able to change my password.
  • I need to be able to recover my password in case I forget it.
  • I need my password to be securely stored and my privacy rights respected.
  • I would like to be able to set security questions.
  • I would like to be able to automatically login without needing to type my credentials again (SSO).
  • I would like to be able to enable Two-Factor Authentication (2FA).
  • I would like to be able to create and modify my profile information.
  • I would like to be able to associate my Google or social media accounts with GameEye and be able to login using them.
  • I would like the option to delete my account.
  • I must not be able to delete other user accounts.
Searching
  • I need to be able to search for games to watch/follow.
  • I need to have my search queries autocompleted by the application
Game Watchlist & New Game Content
  • I need to be able to add games to my watchlist to stay updated about them.
  • I need to see all the recent news articles about each game in my watchlist.
  • I need to see all the recent game updates about each game in my watchlist.
  • I need to see all the recent official Tweets about each game in my watchlist.
  • I need to see all the recent posts on the official subreddits about each game in my watchlist.
  • I need to see a collection of aggregated images about each game in my watchlist.
  • I need to see a collection of aggregated official videos about each game in my watchlist.
  • I need to be redirected to the source site upon clicking/touching an aggregated element (i.e. news article, Tweet, etc.).
  • I would like to see new game content organized in categories.
  • I would like aggregated news articles to show the name of the source and how long ago it was published.
  • I would like aggregated news articles to include a thumbnail image associated with the content if one exists.
  • I would like aggregated Tweets to include a thumbnail image associated with the content if one exists.
  • I would like aggregated Reddit posts to include a thumbnail image associated with the content if one exists.
  • I would like aggregated videos to include a thumbnail image associated with the content if one exists.
  • I would like to have the option to see archives of past video game content (such as older news articles, Tweets, etc.).
  • I would like to be able to determine how old past news must be before being hidden.
  • I would like to see a ranking of the most watched games on the platform, which I can use to find out about potentially good games.
Notifications
  • I need to receive push-notifications about new video game content.
  • I need to see a UI count of how much new content is available for each game in my watchlist.
  • I need to see a UI count of how much new content for each category is available for each game in my watchlist.
  • I need to see push notifications across all the platforms where I have the application installed.
  • I need to be able to submit feedback and crash data from my device to help resolve issues.
  • I would like to be able choose the content categories to be notified about (i.e. news articles, game updates, Tweets, Reddit posts, etc.).
  • I would like to choose how often to receive notifications (i.e. daily, on specific days, or weekly).
  • I would like to be notified about suggested video games to follow/watch.
  • I would like to customize where I receive notifications (i.e. email, push notifications).

Tester

Privileges
  • I would like to be able to do everything a guest can do.
  • I would like to be able to do everything a registered user can do.
Database Simulation
  • I need to be able to change today's date to see how well the application works across different times.
  • I need to be able to clean the database of simulated and temporary test data.
  • I need to be able to simulate a main database connection failure.
  • I need to be able to simulate a video game database connection failure.
  • I need to be able to prepopulate the database with simulated/mock data.
  • I need to be able to simulate the release of a new article, Tweet, Reddit post, or other content type for the database.
  • I need to view recent articles and test data stored in the database.
  • I need to access an archive of the simulated data.
  • I would like to be able to change to language to see how well the application works and looks across different languages.
Cross-Platform Simulation
  • I need to simulate different operating systems to make sure functionality is intact (i.e. Android/iOS, Windows, Linux, macOS).
  • I need to simulate different browsers to make sure functionality is intact as a progressive web application.
Notification Simulation
  • I need to simulate push-notifications to the user.
  • I need to simulate notifications from different content categories.
  • I need to simulate cross-platform notification over different devices.
  • I would like to modify how often notifications are sent to me for testing purposes.
Watchlist Simulation
  • I need to be able to add a game from the mock database to my watchlist.
  • I need to remove a game from my watchlist.
  • I need to be able to simulate the release of a new article, Tweet, Reddit post, or other content type for a game in my watchlist.
  • I need to create a mock news article to link to my watchlist.
  • I need to set the thumbnail image for a mock news article.
User Simulation & Application Data
  • I need to be able to view metrics about the backend and frontend resource utilization (CPU, RAM, disk, etc.).
  • I need to be able to change user settings in a test environment.
  • I need to simulate the creation of a new account as a guest.
  • I need to be able to effectively navigate between different pages within the application.
  • I need to be able to switch between being a guest and registered user.
  • I need to simulate a login for a registered user.
  • I need to be able to mock user interaction with the interface.
  • I must not be able to delete actual user accounts.

Administrator/Developer

Privileges
  • I need to be able to do everything a guest can do.
  • I need to be able to do everything a registered user can do.
  • I need to be able to do everything a tester can do.
Account Management
  • I need to be able to add a new user account.
  • I need to be able to delete user accounts.
  • I need to be able to control user accounts within reason.
  • I need to be able to suspend user accounts.
  • I need to be able to reactivate user accounts (in case they were suspended for any reason).
  • I must not be able to delete user accounts without documented reason.
Game Content
  • I need to be able to add new video game content categories.
  • I would like to be able to suppress/hide certain aggregated video game content (e.g. because of profanity, ESRB rating)
Statistics & Application Information
  • I need to be able to see analytics and usage statistics about the application.
  • I need to be able to receive and view error/crash reports from user devices.
Development Pipeline
  • I need to be able to easily deploy code changes and see the changes reflected immediately on user devices (possible because of PWA).
  • I need to have continuous integration that runs automated tests on the codebase whenever a merge on master or otherwise a primary development branch occurs.
Infrastructure & Database Management
  • I need to be able to directly access and modify the database.
  • I need to be able to directly access infrastructure servers.
  • I need to able to manage load balancers and easily horizontally scale servers (backend, database, etc.) if necessary.
  • I need to be able to set user account permissions.

Risk Matrix

Technical Risks:
  • T1: The structure of source websites changes, causing web scraping to fail.

  • T2: Video game database connectivity failure.

  • T3: Platform becomes so popular that the servers get flooded.

  • T4: Main database (GameEye database) fails.

  • T5: Data breach exposes user information (in our case only emails and password hashes).
Technical Risk Mitigations:
  • T1: Scrape RSS feeds, which are far less likely to change.

  • T2: Store video game database content on our own database for redundancy or build our own video game database, but the latter is difficult.

  • T3: Make solution scalable; use load balancers, have multiple backend servers and multiple database servers.

  • T4: Cache content on user's device so that the application does not appear blank; have more instances of the main database for redundancy.

  • T5: Store less sensitive information; use database encryption; use 3rd party authentication provider to transfer some risk to them.
Customer Risks:
  • C1: Gamers may believe that they do not need GameEye in order to follow news of games that interest them, as they may follow their favorite games on social media or can look up any game they want on Google to get news.

  • C2: Those that run video game news websites may believe that GameEye is trying to do their job for them.

  • C3: User does not know how to use the application.

  • C4: User does not like how information is categorized.
Customer Risk Mitigations:
  • C1: GameEye provides features that its competition may lack, such as noise filtering and the ability to track games.

  • C2: GameEye is not meant to be a replacement for video game news websites, as such, it provides links to already existing news sources which increases traffic to them.

  • C3: Provide instructional material (in-app tutorial, videos, etc.)

  • C4: Provide the ability for users to create their own categories; this would be a premium feature under a paid user plan because it would be expensive for us otherwise.

Security Risks:
  • S1: NoSQL Injection; Malicious NoSQL queries could be injected into the application via improperly handled input data on the client to run on the database. NoSQL injections be used to modify data or execute operations on the database.

  • S2: Cross-Site Scripting (XSS); Malicious scripts could be injected into the browser to redirect users to potentially malicious sites and steal data.

  • S3: Data Exposure; Sensitive user data could be improperly transmitted or stored, allowing for potential exposure.

  • S4: Broken Authentication and Session Control; Poor authentication and session control can allow attackers unauthorized access to user accounts (including admin accounts) through means such as compromising user passwords or session keys.

  • S5: Broken Access Control; Poor access control can allow users access outside of their intended permissions, which can lead to unauthorized data modification and functionality.
Security Risk Mitigations:
  • S1: Queries used to operate and build the GameEye database will not be built by concatenating strings and will utilize prepared statements and query-building mechanisms provided by MongoDB.

  • S2: GameEye will properly sanitize and validate user inputs.

  • S3: Sensitive user data stored within the GameEye database will be encrypted both when stored and in transit.

  • S4: Proper password precautions and requirements will be in place for account login alongside multi-factor authentication; session IDs will also not be stored in the URL.

  • S5: Access control will be enforced server-side where attackers can't access alongside using integration tests to ensure users can't access specific URLs without proper authentication.
